# 小龙虾启动器新手使用文档

本文面向第一次使用 `openclaw-launcher` 的用户，尽量用最直接的方式说明每个页面能做什么、该怎么操作，以及哪些地方需要特别注意。

## 1. 这个启动器是做什么的

小龙虾启动器是一个桌面工具，用来管理和启动 OpenClaw 的本地数据。

你可以把一只“龙虾”理解成一套独立的 OpenClaw agent 数据。启动器主要提供这些能力：

- 管理默认本地龙虾和导入进来的龙虾
- 启动指定龙虾对应的 OpenClaw
- 打开当前龙虾的控制网页
- 浏览这只龙虾的设定文档、技能、定时任务、记忆和账号信息
- 查看聊天记录并继续对话
- 查看龙虾主动发来的通知和提醒
- 直接查看和编辑 `README.md`
- 导入和导出 `.claw` 龙虾包
- 管理启动路径、本地目录和连接服务设置

## 2. 使用前准备

第一次打开后，建议先做这三件事：

1. 进入“设置”
2. 确认 `OpenClaw 启动入口`
3. 确认 `默认本地龙虾目录`

如果你不清楚路径在哪里，优先点击“自动识别”。识别结果会列出候选项，你可以直接点“使用这个结果”。

如果自动识别不对，再手动使用：

- “选择启动入口”：选择 OpenClaw 的可执行文件；在 macOS 上可以选择 `.app` 或实际启动入口
- “选择本地目录”：选择默认本地龙虾所在的数据目录
- “保存设置”：保存当前修改

只有当启动入口和本地目录都正确时，启动器才能正常启动龙虾。

## 3. 界面总览

左侧导航分为 6 个页面：

- 首页
- 龙虾
- 聊天
- 通知
- 文档
- 设置

左侧栏还会显示：

- 当前是否已完成基本设置
- 当前使用的是哪只龙虾
- 当前是否可以启动

## 4. 首页

首页适合日常快速启动。

你可以在这里做 3 件事：

- 点击“启动龙虾”：启动当前选中的龙虾
- 点击“切换龙虾”：跳转到“龙虾”页面选择别的龙虾
- 点击“打开控制网页”：打开当前龙虾对应的控制页面

首页还会显示最近一次启动时间，方便你确认上次运行记录。

### 启动时会发生什么

启动龙虾时，启动器会：

- 按当前配置找到 OpenClaw 启动入口
- 根据当前龙虾切换到对应 profile 或状态目录
- 检查连接服务是否可用
- 如有需要，在后台启动这只龙虾自己的连接服务
- 记录最近启动历史

如果当前路径没配好，或者启动入口文件已经不存在，启动会失败，这时先去“设置”检查路径。

## 5. 龙虾页面

“龙虾”页面是整个应用的管理中心。

### 5.1 左侧龙虾列表

这里会显示：

- 默认本地龙虾
- 所有已导入的龙虾

你可以：

- 点击任意龙虾卡片查看详情
- 在搜索框里按名称或路径搜索
- 点击“刷新列表”重新扫描

默认本地龙虾代表你当前电脑上的主数据目录，不是导入包，所以不能删除，也不能改名。

### 5.2 右侧详情区

选择一只龙虾后，右侧会显示：

- 名称
- 目录
- 来源
- 创建时间
- 最近启动时间

如果是导入的龙虾，还可以：

- “启动”
- “改名”
- “删除”

删除前会弹确认框。删除后无法恢复。

### 5.3 内容盘点区

每只龙虾下面都会按分类显示可浏览的内容。点击“展开”后可查看条目，点击“预览”可直接看内容。

包含 5 类：

- 设定文档
  - 来自 `workspace` 下的标准设定文档
  - 例如 `AGENTS.md`、`BOOTSTRAP.md`、`HEARTBEAT.md`、`IDENTITY.md`、`SOUL.md`、`TOOLS.md`、`USER.md`
- 技能列表
  - 来自 `workspace/skills/*/SKILL.md`
- 定时任务列表
  - 来自 `cron/jobs.json`
- 记忆列表
  - 包括主记忆库 `memory/main.sqlite`
  - 也包括 `agents/main/sessions` 下的会话记忆文件
- 账号列表
  - 包括账号配置、已配对设备和本地身份文件

说明：

- 预览窗口主要用于快速查看
- 很长的文本会被截断显示
- SQLite 记忆库目前只显示文件信息，不会直接解析库内容

### 5.4 导入龙虾包

点击“导入龙虾包”后，启动器会让你选择一个 `.claw` 文件。

导入流程是：

1. 选择 `.claw` 包
2. 启动器先校验压缩包内容
3. 输入导入后的龙虾名称
4. 导入完成后，这只龙虾会出现在列表中

安全校验会检查：

- `manifest.json` 是否存在
- 包内文件是否和清单一致
- 文件大小是否一致
- `SHA-256` 哈希是否一致

如果启动器提示这个包可能被篡改，说明完整性校验没通过。新手默认应该取消导入，只有在你明确知道来源可信、并且理解风险时，才考虑继续。

导入后的龙虾会保存在“设置”里配置的“导入龙虾保存位置”；如果你没有改过，一般会保存到应用默认目录。

### 5.5 导出龙虾包

点击“导出龙虾包”后，会打开导出窗口。

你需要设置：

- 龙虾包名称
- 导出位置

还可以选择两个可选项：

- 导出记忆和历史会话
- 导出账号和设备信息

这两个选项默认关闭，建议新手保持关闭。

原因是默认导出策略更安全：

- 不包含记忆内容
- 不包含历史会话
- 不包含账号登录信息
- 不包含设备配对信息

如果你手动开启这些选项，启动器会再次弹出风险确认，因为这可能泄露敏感数据。

导出完成后，会生成一个 `.claw` 文件，方便备份或在另一台机器导入。

## 6. 聊天页面

“聊天”页面用于查看和继续与当前龙虾的对话。

页面顶部会显示：

- 当前正在使用哪只龙虾
- 当前连接服务状态

连接状态有 3 种常见情况：

- 已连上：可以正常聊天
- 正在连接：连接服务正在启动或还没准备好
- 未连上：当前还无法建立聊天连接

### 6.1 对话列表

左侧是对话列表，你可以：

- “新建对话”
- 选择已有对话
- “重命名”对话
- “删除”对话

对话会按最近更新时间排序。

### 6.2 对话区

右侧是消息内容区。

你可以：

- 查看历史消息
- 输入新消息
- 点击“发送给当前龙虾”

发送后，界面可能出现两种过程状态：

- “处理中”：请求已经发出，正在等待回复
- “生成中”：回复正在流式返回

如果系统环境里缺少某些运行条件，真流式聊天可能不可用，但对话仍可能以非流式方式完成。

## 7. 通知页面

“通知”页面集中显示当前龙虾主动产生的消息，最多保留最新 100 条。

目前主要有两类：

- 定时任务通知
  - 来自 `cron/runs` 中完成的任务记录
  - 会显示任务结果摘要；如果有错误，也会一起显示
- Heartbeat 通知
  - 来自心跳会话
  - 用于展示龙虾主动发来的状态更新或提醒

适合用来查看“不是你主动发起”的消息。

如果这里是空的，表示当前还没有新的主动消息或提醒。

## 8. 文档页面

“文档”页面显示当前选中龙虾目录下的 `README.md`。

你可以：

- 刷新文档
- 编辑
- 保存
- 取消编辑

这适合做一些轻量级说明维护，比如：

- 记录这只龙虾是干什么的
- 记录工作习惯、约束和交接说明
- 给自己或团队写使用备注

如果当前目录里没有 `README.md`，页面会明确提示未找到。你在这里点击“编辑”并保存后，启动器会直接写入一个新的 `README.md`。

## 9. 设置页面

“设置”页面分为基础设置和高级设置。

### 9.1 基础设置

这里最重要，建议优先确认：

- `OpenClaw 启动入口`
- `默认本地龙虾目录`

常用操作：

- “自动识别”：扫描可用安装位置
- “选择启动入口”：手动选可执行文件
- “选择本地目录”：手动选默认数据目录
- “保存设置”：保存当前内容

如果识别到了多个候选项，通常优先选：

- 路径存在
- 数据目录也识别成功
- 明显对应你正在使用的 OpenClaw 安装

### 9.2 高级设置

点击“展开高级设置”后，可以配置更多内容。

#### 龙虾目录

这里的“导入龙虾保存位置”决定了导入的 `.claw` 包解压后存到哪里。

如果你想把导入龙虾集中保存到某个专用目录，可以在这里改。

#### 连接服务

这里可以配置聊天和控制网页依赖的连接服务：

- 连接地址
- 健康检查路径

并提供两个手动按钮：

- “启动连接服务”
- “停止连接服务”

适用场景：

- 你想手动测试连接是否正常
- 自动连接失败，想自己排查
- 你使用的是自定义地址或自定义健康检查路径

对大多数新手来说，不改这里也可以。只有在默认配置无法工作时再动它。

## 10. 常见使用场景

### 场景一：第一次安装后开始使用

1. 打开“设置”
2. 点击“自动识别”
3. 确认启动入口和默认本地龙虾目录
4. 点击“保存设置”
5. 回到“首页”
6. 点击“启动龙虾”

### 场景二：把另一台机器上的龙虾带过来

1. 进入“龙虾”
2. 点击“导入龙虾包”
3. 选择 `.claw` 文件
4. 等待校验完成
5. 输入名称
6. 导入后在列表中选中它并启动

### 场景三：导出一只龙虾给别人

1. 进入“龙虾”
2. 选中要导出的龙虾
3. 点击“导出龙虾包”
4. 设置包名和导出目录
5. 保持敏感选项关闭
6. 点击“确认导出”

### 场景四：只想快速查看这只龙虾里有什么

1. 进入“龙虾”
2. 选中目标龙虾
3. 展开“设定文档”“技能列表”“定时任务列表”“记忆列表”“账号列表”
4. 点击“预览”查看内容

### 场景五：继续之前的聊天

1. 进入“聊天”
2. 确认顶部显示的是你想使用的龙虾
3. 在左侧选已有对话，或新建对话
4. 在右侧继续发送消息

## 11. 安全与注意事项

请特别注意下面几点：

- 导入 `.claw` 包时，如果提示包被篡改，默认不要继续
- 导出时，除非你明确知道自己在做什么，否则不要开启“导出记忆和历史会话”
- 同样地，不要随意开启“导出账号和设备信息”
- 删除导入龙虾不可恢复，删除前先确认不再需要
- 默认本地龙虾代表本机正在使用的数据，不建议随意改目录
- 聊天、通知、控制网页都依赖连接服务；如果这些功能异常，优先检查“设置”中的连接服务配置

## 12. 遇到问题先看哪里

### 无法启动龙虾

先检查：

- 设置里的启动入口是否正确
- 启动入口文件是否还存在
- 默认本地龙虾目录或导入龙虾目录是否存在

### 聊天页显示“未连上”

先检查：

- 这只龙虾是否已经成功启动
- 设置里的连接地址和健康检查路径是否正确
- 高级设置里手动启动连接服务后是否恢复

### 导入失败

先检查：

- 文件是否是 `.claw`
- 压缩包是否损坏
- 是否被完整性校验判定为篡改

### 文档页看不到 README

这通常表示当前龙虾目录下没有 `README.md`。你可以直接在文档页点击“编辑”后保存，启动器会创建它。

## 13. 一句话总结

如果你只记住最基本的操作顺序，可以按这个来：

先在“设置”配好路径，再在“首页”或“龙虾”里启动；需要看内容去“龙虾”和“文档”，需要继续聊天去“聊天”，需要看主动消息去“通知”，需要迁移或备份就用“导入/导出龙虾包”。
